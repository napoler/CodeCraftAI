# CodeCraft Spec: AI 规约驱动开发框架

> **[阅读我们的核心原则 (`PRINCIPLES.md`)](./PRINCIPLES.md)**

欢迎来到 **CodeCraft Spec**，一套为 `CodeCraftAI` 项目量身打造的、以“**架构级上下文工程**”为最高原则的官方开发框架。

## 核心理念

本框架的理念是，通过一套严格的、文档优先的流程，来指导和约束 AI（及人类）的开发行为。我们不仅仅是在定义单个功能，我们是在**构建一个自维护的、代码与文档一体化的项目知识图谱**。

每一个开发任务都始于一份深度结构化的**规约 (Spec)**。这份规约不仅定义了功能本身，更重要的是，它将功能作为一个新的“知识节点”，注册到了项目的全局索引之中。反过来，AI 在开启新任务前，也必须优先加载和使用这份索引。

## 如何使用？

1.  **首先，请务必阅读并理解我们的 [核心原则 (`PRINCIPLES.md`)](./PRINCIPLES.md)。**
2.  然后，请学习我们的 [规约语法 (`SYNTAX.md`)](./SYNTAX.md)。
3.  最后，请遵循我们的 [工作流 (`WORKFLOW.md`)](./WORKFLOW.md) 来开始您的第一个规约。

通过 `CodeCraft Spec`，我们致力于将 AI 从一个“代码工具”提升为一个真正的“工程伙伴”，一个能够理解和运用整个项目上下文的智能体。
