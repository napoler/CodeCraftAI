# 语义标签指令系统指南 (Semantic Tag Directives Guide)

## 1. 目的 (Purpose)

本指南详细介绍了 CodeCraftAI 项目中内置的一套强大的**语义标签指令系统**。这套系统允许用户通过**自由定义**的XML风格标签，对 AI 的写作行为进行极其灵活和精确的程序化控制，将AI协作提升到一个全新的维度。

## 2. 核心理念：从命令到语义 (Core Philosophy: From Commands to Semantics)

本系统的核心思想是：用户不再局限于使用预设的命令，而是可以通过**标签的名称本身**，向 AI 传达复杂的**意图 (Intent)** 和 **上下文 (Context)**。

AI 的核心任务是**智能地解释每个标签的语义**，并将标签内的内容作为该语义下的最高优先级指令。

---

## 3. 指令详解 (Directives Explained)

### 3.1. 自由定义的通用语义标签 (User-Defined General Semantic Tags)

*   **语法:** `<标签名>...</标签名>`
*   **定义:** 用户可以根据任务需要，**自由创建任何标签名**来包裹和定义信息。标签名应使用清晰、表意明确的词语。
*   **AI 行为:**
    *   AI **必须**智能地理解标签名的语义，并严格遵循其所定义的约束或指令。
    *   所有自定义标签都具有高优先级，AI 会将其作为指导其思考和创作的核心框架。
    *   所有标签本身**绝不能**出现在最终交付的文章正文中。

*   **示例库 (Example Library):**

    *   **风格与基调控制:**
        ```xml
        <写作基调>专业、严谨但略带一丝风趣</写作基调>
        ```
    *   **核心论点定义:**
        ```xml
        <核心论点>尽管陶瓷内胆更健康，但其耐用性是用户需要权衡的主要成本。</核心论点>
        ```
    *   **受众画像:**
        ```xml
        <目标受众>对健康生活有追求，但对厨具技术细节不太了解的年轻家庭。</目标受众>
        ```
    *   **负面约束 (内容排除):**
        ```xml
        <必须避免的词语>廉价, 入门级</必须避免的词语>
        <禁止讨论的话题>竞争对手的具体型号</禁止讨论的话题>
        ```

### 3.2. `<资料>` - 受保护的“事实之源”标签 (The Protected "Source of Truth" Tag)

*   **语法:** `<资料>...</资料>`
*   **定义:** 在所有用户可自由定义的标签中，`<资料>`是一个**特殊的、受系统保护的保留标签**。它的作用是唯一的：**定义本次任务中不可更改的、绝对优先的核心事实基础**。
*   **AI 行为:**
    *   无论用户定义了多少其他标签，AI 都**必须**将`<资料>`标签内的所有信息视为“**真理**”。
    *   AI **不得**对此标签内的信息进行任何事实性的修改、质疑、反驳或补充。
    *   当其他指令（例如 `[...]` 补全）需要事实依据时，必须**优先**从此标签内的内容中寻找。

### 3.3. `[...]` 和 `<...>` - 通用自主补全 (Universal Autonomous Completion)

*   **定义:** 这两种指令的功能**完全相同**。它们都是占位符，要求 AI 自主地、高质量地完成补全任务。
*   **AI 行为:**
    *   AI 会智能地**综合利用所有上下文**来进行补全，包括用户提供的所有**语义标签**（特别是`<资料>`）、括号内的提示以及它自身的背景知识。
    *   **示例:**
        ```xml
        <资料>
        产品名称: "Model-Z 咖啡机"
        冲煮温度: 95°C
        </资料>
        <写作基调>轻松愉快</写作基调>

        <...> " — Model-Z 咖啡机让您的每个早晨都充满活力！它能在 [...] 的精确温度下为您萃取完美咖啡。"
        ```
        *   在上述例子中，AI 会根据`<写作基调>`生成一段轻松的开场白填充到`<...>`，并根据`<资料>`中的事实填充`[...]`为`95°C`。

## 4. 最佳实践

*   **组合使用标签:** 语义标签的威力在于组合。通过同时定义`<核心论点>`、`<目标受众>`和`<写作基调>`，您可以像导演一样，为 AI 设定一个极其清晰的创作场景。
*   **明确优于模糊:** 标签名越是清晰、无歧义，AI 的执行效果就越好。
*   **安全第一:** 始终将最核心、不可变的事实放入`<资料>`标签中，这是保证内容准确性的最佳实践。
