# 内容表现复盘与规范迭代规范 (Content Performance Review & Spec Iteration Profile)

## 1. 核心理念 (Core Philosophy)

*   **基础规范继承 (Standard Inheritance):** 本文档遵循 `specs/ai-writing-standard.md` 中定义的所有通用原则。
*   **核心任务 (Core Task):** 定义一个“学习闭环”，指导AI基于已发布文章的真实表现数据，进行深度分析，并提出针对现有写作规范的具体、可执行的优化建议（以`git diff`格式），从而驱动整个内容体系的持续进化。

## 2. 输入 (Inputs)

*   **[被分析文章链接]:** 指向需要复盘的已发布文章的URL。
*   **[关联写作档案]:** 创建该文章时所遵循的“写作档案”路径 (例如: `specs/seo-content-revision.md`)。
*   **[表现数据]:** 一个包含以下指标的结构化数据集：
    *   `page_views`: 页面总浏览量。
    *   `avg_time_on_page`: 用户平均页面停留时长 (秒)。
    *   `keyword_rank_fluctuation`: 核心关键词排名的变化情况 (例如: 从 "未排名" -> "第15名")。
    *   `bounce_rate`: 页面的跳出率 (%)。
    *   `user_feedback`: 一个包含用户评论或反馈文本的列表。
*   **[竞品文章链接 (可选)]:** 1-2篇表现优异的竞争对手文章链接。

## 3. 智能工作流 (Intelligent Workflow)

### 阶段一：数据摘要与初步诊断 (Phase 1: Data Summary & Initial Diagnosis)

1.  **数据结构化呈现:**
    *   **指令:** 将所有输入的`[表现数据]`以清晰的Markdown表格形式呈现。

2.  **初步诊断:**
    *   **指令:** 基于数据，给出一个关于文章整体表现的初步、高级别的诊断 (例如：“流量指标良好，但用户停留时长低于预期，可能表明内容深度不足”)。

### 阶段二：深度归因分析 (Phase 2: In-depth Root Cause Analysis)

*   **指令:** AI必须逐一、深入地回答以下所有核心问题，以探究数据背后的根本原因。

1.  **“蓝海切入点”有效性分析:**
    *   文章最初选择的“蓝海切入点”是否成功？`keyword_rank_fluctuation` 数据是否表明我们有效占领了一个低竞争的 niche？

2.  **“知识性”与“价值证明”检验:**
    *   `avg_time_on_page` 和 `user_feedback` 数据是否证明了文章提供了足够深度的、有价值的知识？用户是真的在消费内容，还是快速离开？

3.  **竞品对比分析 (如果提供):**
    *   与`[竞品文章链接]`相比，我们的文章在数据上表现如何？从竞品的成功或失败中，我们可以学到什么？

4.  **原创贡献价值分析:**
    *   文章中提出的原创观点或独特类比，是否在`user_feedback`中得到了正面或负面的印证？

### 阶段三：规范优化建议 (Phase 3: Specification Improvement Proposal)

*   **指令:** 这是本规范最核心的步骤。基于以上所有分析，AI必须提出针对`[关联写作档案]`的具体、可执行的修改建议。

1.  **定位问题:**
    *   明确指出当前规范中的哪一条或多条指令，是导致内容表现未达预期的根本原因。

2.  **生成修改补丁:**
    *   **指令:** AI必须以标准的`git diff`格式，生成一个可以直接应用于`[关联写作档案]`文件的修改补丁。
    *   **要求:**
        *   补丁必须是完整、语法正确的。
        *   必须包含清晰的注释，解释为什么要做此项修改。

## 4. 输出结构 (Output Structure)

*   **[文件名]:** `review-{{被分析文章的唯一标识符}}.md`
*   **[阶段1成果: 数据摘要与初步诊断]**
    *   表现数据表格: ...
    *   初步诊断结论: ...
*   **[阶段2成果: 深度归因分析]**
    *   蓝海切入点有效性分析: ...
    *   “知识性”与“价值证明”检验: ...
    *   竞品对比分析: ...
    *   原创贡献价值分析: ...
*   **[阶段3成果: 规范优化建议]**
    *   **定位到的问题指令:** ...
    *   **规范修改建议 (Git Diff格式):**

        +++ b/specs/seo-content-revision.md
        @@ -XX,X +XX,X @@
        - (旧的指令)...
        + (经过优化的新指令)...
