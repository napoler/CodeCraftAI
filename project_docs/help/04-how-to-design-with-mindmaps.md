# 帮助文档 04：如何使用脑图做设计

## 1. 为什么设计要用脑图？

在动手写 Spec 或代码之前，我们需要一个更高层次的视角来梳理功能的整体结构。脑图 (Mind Map) 是完成这项任务的完美工具。

- **结构化思考**: 脑图能帮助我们将一个复杂的功能，逐层分解成更小的、可管理的模块和子任务。
- **可视化**: 它能将功能的全局视图直观地展示出来，方便团队快速理解和评审。
- **激发创意**: 在脑图上自由地添加和移动节点，有助于激发新的想法和发现潜在的问题。

## 2. 我们的脑图工作流

我们推崇一种“模板化、代码化”的脑图设计流程。

### a. 找到模板

我们项目内置了一个通用的功能设计脑图模板：
`.codecraft/templates/mindmap_template.md`

这个模板是一个纯文本的 Markdown 文件，预设了一个典型 Web 应用功能的结构，涵盖了前端、后端和非功能性需求。

> **为什么要用纯文本的 Markdown？**
> 因为它能被 Git 完美地进行版本控制，AI 也可以轻松地读取、理解甚至为你自动生成初步的脑图。

### b. 开始你的设计

1.  **复制模板**: 当你准备设计一个新功能时（例如，一个“订单管理系统”），请**不要**直接修改模板文件。你应该将它**复制**到 `project_docs/` 目录下。

    ```bash
    # 示例命令
    cp .codecraft/templates/mindmap_template.md project_docs/order-management-system-design.md
    ```

2.  **编辑脑图**: 打开你新创建的 Markdown 文件，开始进行修改和填充：
    - 将顶层的标题改成你的功能名称。
    - 根据你的实际需求，添加、删除或修改各个节点。
    - 使用 Tab 或空格来创建和调整层级关系。

### c. 评审与归档

- 设计完成后，你可以将这份 Markdown 脑图链接发给团队成员进行评审。
- 一旦设计确定，这份文档就成为了该功能的核心设计蓝图，存放在 `project_docs/` 中，与你的其他项目文档一起归档。

## 3. 使用图形化工具

如果你和你的团队更喜欢使用可视化的脑图工具（如 XMind, Miro, Draw.io 等），完全没有问题！

我们的工作流程同样支持：

1.  使用你喜欢的工具进行设计。
2.  设计完成后，将脑图导出为一张图片（推荐 `.png` 或 `.svg` 格式）。
3.  将导出的图片文件，同样存放在 `project_docs/` 目录下，并确保其命名清晰易懂（例如 `order-management-system-design.png`）。

**最佳实践**: 即使你使用了图形化工具，我们仍然推荐你复制一份 `mindmap_template.md`，并在其中写下一些核心的设计要点，然后链接到你的图片文件。这能让你的设计同时具备文本的可搜索性和图形的可视化优势。
