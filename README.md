# CodeCraftAI
Develop an AI-assisted programming tool that automates code generation, optimizes existing code, and provides real-time debugging suggestions, aiming to improve developers' coding efficiency while ensuring code quality and maintainability.

---

## 快速使用指南 (Quick Start)

本节将指导你如何将我们的 **规范驱动开发 (Spec-Driven Development)** 流程快速集成到你自己的项目中。

### 手动集成

适用于希望手动操作的开发者。

1.  **复制规范目录**: 将本仓库中的 `specs` 目录完整地复制到你的项目根目录下。
2.  **开始使用**:
    *   当需要开发新功能或进行重要变更时，参考 `specs/README.md` 的流程说明。
    *   复制 `specs/template.md` 文件来创建你的第一份规范。

### 使用AI助手自动集成

你可以使用以下提示词 (Prompts) 指导AI编程助手（如 GitHub Copilot, Cursor, Codeium 或其他类似工具）为你自动完成集成。

#### 场景1：在新项目中初始化

**目标**: 在一个全新的项目开始时就建立规范流程。

**提示词**:
```
任务：为我的新项目初始化“规范驱动开发”流程。

上下文：我正在启动一个新项目，希望从一开始就采用规范驱动开发（Spec-Driven Development）的最佳实践。我需要你为我创建支持这一流程所需的基础文件结构和模板。

请执行以下操作：
1.  在项目的根目录下创建一个名为 `specs` 的文件夹。
2.  在 `specs` 文件夹内，创建一个 `README.md` 文件，用于向团队解释规范驱动的流程、目的和生命周期（草案 -> 评审 -> 批准 -> 实现 -> 完成）。
3.  同样在 `specs` 文件夹内，创建一个名为 `template.md` 的规范模板文件。这个模板需要包含以下几个核心部分，并为每个部分提供引导性的注释或问题：
    - 规范名称、状态、负责人等元数据
    - 目标 (Goals)
    - 背景 (Background)
    - 设计方案 (Proposed Solution)
    - 备选方案 (Alternatives Considered)
    - 测试计划 (Test Plan)

请确保生成的文件内容是专业、清晰且随时可用的。完成后，告诉我文件已创建成功。
```

**提示词解析**:
*   **角色和任务**: 清晰地定义了AI需要扮演的角色（项目初始化助手）和要完成的具体任务。
*   **上下文**: 解释了“为什么”需要做这件事，帮助AI理解最终目标，从而产出更相关的结果。
*   **明确的指令**: 使用了数字列表，将一个大任务分解为多个具体、可执行的小步骤。
*   **定义产出标准**: 要求内容“专业、清晰且随时可用”，为AI的输出设定了质量标准。

#### 场景2：在现有项目中集成

**目标**: 将规范流程平稳地引入一个已在进行中的项目。

**提示词**:
```
任务：将“规范驱动开发”流程集成到我现有的项目中。

上下文：我的项目已经进行了一段时间，现在团队希望引入规范驱动开发（Spec-Driven Development）流程来提高沟通效率和代码质量。我需要你帮助我在不扰乱现有结构的情况下，添加必要的规范文件。

请在当前项目的根目录下执行以下操作：
1.  创建一个 `specs` 文件夹。
2.  在该文件夹内创建 `README.md` 文件，用于解释规范的生命周期和使用方法。
3.  在该文件夹内创建 `template.md` 文件，作为编写新规范的标准模板，需包含目标、背景、设计方案、备选方案和测试计划等部分。

请直接为我创建这些文件和它们的内容。
```

**提示词解析**:
*   **针对性调整**: 这个提示词与上一个类似，但微调了上下文，强调了是在“现有项目”中操作，并要求AI“不扰乱现有结构”，这让指令更加安全和精确。
*   **直接行动**: 指令更侧重于“直接创建”，因为对于现有项目，我们通常希望AI能快速、准确地完成文件添加，而不需要过多的交互。
